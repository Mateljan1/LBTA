# Coach 1:1 Talking Points

> 15-minute 1:1 between Andrew and a coach (Allison or Saska). Use these talking points when adopting feels stuck, voice is drifting, or trust is low.

---

## When to run a coach 1:1

- Coach hasn't used the GPT in 5+ days.
- Coach said "it doesn't sound like me" twice in two Fridays.
- A draft was sent that wasn't a fit (small but real).
- Coach is overloaded and the GPT could be relieving them but isn't.
- Coach is enthusiastic and Andrew wants to compound that energy into a documented pattern.

## The frame (don't skip this)

This is **not a performance review.** It's a tool review. The GPT works for the coach, not the other way around. If it's not pulling weight, that's a tool problem, not a coach problem.

Open with: *"Quick one. I'm checking how the GPT is feeling, not how you're using it."*

## The 5 questions (in this order)

### 1. "What's a draft from this past week that felt good?"

Open with strength. Get a specific. Hold the moment.

If they can't think of one: *"Okay, that's the most useful answer. The GPT didn't earn its keep last week. Let's figure out why."*

### 2. "What's a draft that didn't feel right?"

Pull the exact draft if possible. Read it aloud together. Ask:
- What's the one phrase that's off?
- What would *you* have said instead?
- What's the rhythm of *your* version?

This is research for the GPT's voice samples and the system prompt.

### 3. "If the GPT could do one more thing for you, what would it be?"

Don't promise. Listen.

Common asks:
- "Draft replies in the voice of a specific past email I have."
- "Pre-flag the emotionally charged emails before I read them."
- "Cite the policy with the exact line number / link."
- "Stop using exclamation points."
- "Be more specific about the kid / member, less general."

### 4. "What's something you're spending time on that the GPT *isn't* helping with?"

This is the highest-leverage question. The unmet need is where compounding still has runway.

Examples that have come up:
- "Writing the weekly newsletter section for adult clinic — currently I write it from scratch every week."
- "Drafting condolence / care messages when a member is dealing with something hard."
- "Reading and triaging Slack-equivalent inbound parent messages."

If the answer is a real pattern: schedule a build session. Don't promise — *"I'll see what I can do for next week."*

### 5. "What would make this easier for you to use?"

Logistics, not concept. Examples:
- "It needs to be the first tab when I open ChatGPT."
- "Make a phone shortcut."
- "Send me the prompt I should paste before each draft so I don't blank."

These are 5-minute fixes. Do them on the spot.

## After the 1:1

- Update the GPT (system prompt patch / new voice samples / etc.) **before the end of the day.**
- Text the coach: *"Here's what I changed: ___. Try it tomorrow on a real draft. Let me know."*
- Log in `09-COMMAND-CENTER/changelog.md`.

## Patterns to watch

| Pattern | What it usually means |
|---|---|
| "Voice is too founder-y" | Voice samples are stale. Pull 5 fresh ones. |
| "Voice is too generic" | The system prompt has weak voice anchors. Add 2 specific example sentences. |
| "It's faster to write from scratch" | Coach hasn't found the right workflow yet. Walk through 1 use case live. |
| "I forget to use it" | Trigger isn't tight. Make it the first browser tab, or pin it. |
| "It made up a fact" | Knowledge file gap. See `runbooks/hallucination-incident.md`. |
| "I love it" | Don't move on too fast. Document what they love and feed it back into the prompt as a positive example. |

## What to do if a coach decides to stop using the GPT

That's allowed. They own their work.

But: ask why, write it down, and don't push back. The system serves them, not the other way around. If they pick it back up later, great. If not, the GPT was the wrong tool for that coach. Adjust expectations.

This is rare if the build is right. If it happens twice, the build has a bigger problem than that coach can name.
