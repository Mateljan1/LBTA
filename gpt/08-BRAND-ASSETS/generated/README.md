# Generated Brand Assets — Drop Zone

> AI-generated brand-adjacent assets the Founder GPT produces during day-to-day work end up here for review before they're either promoted to canonical assets, or discarded.

## What goes here

- AI-generated mockups, hero variants, social card drafts, profile picture iterations.
- Anything Andrew (or the Founder GPT) creates with Image Generation that might become a canonical asset.

## What does NOT go here

- Final production assets — those live in `/public/` or `/assets/` in the repo root.
- Member or player photographs — never AI-generate these.
- Marketing assets that haven't been reviewed by Andrew.

## Lifecycle

1. **Generated:** Image lands here with a descriptive filename (e.g. `2026-04-25-hero-spring-campaign-v1.png`).
2. **Reviewed:** Andrew reviews against the brand kit. Either:
   - **Promote:** Move to `/public/images/` or wherever it'll be used. Update this folder's index if maintained.
   - **Iterate:** Generate v2 with notes, drop next to v1.
   - **Discard:** Delete. Don't keep weak drafts that compound into "good enough."
3. **Promoted assets:** Removed from this folder once they live somewhere canonical.

## Naming convention

`YYYY-MM-DD-purpose-version.ext`

Examples:
- `2026-04-25-junior-camp-flyer-v1.png`
- `2026-05-02-instagram-grid-may-v3.jpg`

## House rules

- **No real-player imagery.** Even synthetic/composed. The Founder GPT will refuse, but if any image generation tool ever produced one, do not save it here.
- **Brand colors only.** If a generated image uses off-palette colors, regenerate or discard.
- **No text-on-image marketing copy as final.** AI is mediocre at typography. Use it for backgrounds; layer text in Figma.
- **Cap this folder at 50 files.** If it grows beyond, prune. Don't accumulate "maybe later" drafts.

## Owner

Andrew. The Founder GPT can suggest cleanups; only Andrew deletes.
