## What v1.4 fixed (post-Review pass)

After v1.3 shipped, 5 parallel review agents flagged real issues. v1.4 addressed them:

1. **Tightened the eyebrow scan regex** ŌĆö now catches `tracking-widest` / `tracking-wider` / `tracking-wide` / `tracking-tight` / `tracking-tighter` named utilities (previously only `tracking-[ŌĆ”]` arbitrary values), template-literal JSX expressions (`className={\`...\`}`), and string-literal classNames in const declarations / ternaries. Excludes button-context strings via `min-h-[48px]` heuristic (the `.cursorrules` 48px touch-target marker).

2. **Wired `legacyLbta` into strict-mode totals** ŌĆö the docs claimed deprecated lbta-* fails strict mode but the implementation didn't enforce. Now matches the docs.

3. **Reverted ~39 button over-migrations to a new `text-button` utility** ŌĆö v1.3's mass migration treated Book/Reserve/Register button labels as eyebrows because the regex matched their shape. Buttons now use `text-button` (0.7rem, 0.18em, weight 500) or `text-button-sm` (0.65rem, 0.16em, weight 600), distinct from labels.

4. **Removed baked font-weight from `.text-eyebrow` Tailwind utility** ŌĆö was 600, silently bolded any callsite that didn't add explicit `font-medium`. Now callers must specify; the v1.4 migration adds `font-semibold` to the 22 newly-surfaced eyebrows.

5. **Deleted duplicate `.text-eyebrow` from `globals.css` components layer** ŌĆö was overridden by the Tailwind utilities-layer version anyway, so dead code with cascade-flip footgun behavior. Single source of truth: `tailwind.config.ts:69-82`.

6. **Deleted dead code**: `brandHex()` + `BRAND_BY_KEBAB` (zero callers), `handRolledSectionPaddingRegex` (zero hits ever).

## Acknowledged visual changes from v1.3 (not regressions, but worth knowing)

The v1.3 migration standardized letter-spacing on every label/eyebrow:
- Tracking values 0.05emŌĆō0.22em (previously hand-tuned per callsite) ŌåÆ 0.18em (canonical)
- Most visible: `UtrWeekAtGlanceBanner` Saturday/Sunday day labels went from 0.05em ŌåÆ 0.18em (3.6├Ś wider)
- Modal form labels: 0.10em ŌåÆ 0.18em (80% wider)

This is a **system-thinking decision** (one canonical eyebrow tracking is what the design system is for), not a bug. The user-visible delta is real but small. If any specific callsite needs different tracking, override at the callsite (e.g. `text-eyebrow tracking-wider`) ŌĆö Tailwind utilities win over the size-utility's baked spacing.
