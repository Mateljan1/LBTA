**Date:** 2026-05-11
**Workflow:** `/compound:full` wrapped with `/agent-architecture` v2.0
**Autonomy:** Level 4 ‚Äî execute end-to-end, no mid-run interruptions
**Predecessor:** `plans/2026-05-11-lesson-plan-library-plan.md` (shipped earlier today)

## Overview

Form-driven deterministic lesson plan composer at `/coach-hub/{coach}/lesson-plans/generate`. Coach picks program + age + duration + focus; pure-logic composer assembles a 4-block plan from the existing `framework.json` libraries. Generated plans persist in **Vercel KV** (already configured) and appear as a "Generated drafts" section on the library landing alongside the 12 curated starter plans.

## Problem Statement

Library has 12 curated plans. When a coach wants a Green Ball Volley Day for 9-10yo (no exact match), they have to mentally adapt the closest plan. The framework + libraries we already codified in `lesson-plan-types.ts` BLOCK_TYPES + `framework.json` warmupLibrary/craftLibrary have everything needed to generate-on-demand. Unblock the planning friction without inventing new content.

## Proposed Solution

**Pure-logic deterministic composer** (no AI). Drives directly off the canonical framework. Output is a `LessonPlan` shape that reuses the existing `LessonPlanDetailView` renderer.

**Routes:**
- `/coach-hub/{coach}/lesson-plans/generate` ‚Äî form
- `/coach-hub/{coach}/lesson-plans/drafts/{draftId}` ‚Äî generated plan detail
- `/api/coach-hub/{coach}/lesson-plans/generate` ‚Äî POST: compose + persist + return draftId

**Composer logic** (`lib/lesson-plan-composer.ts`):
1. **Movement Warm-up** ‚Üí pick from `framework.warmupLibrary.byAge[bandFor(ageBand)]` + universalOpener
2. **Craft Block** ‚Üí pick from `framework.craftLibrary.strokeFundamentals` (matched on focus) OR `tactical.items` if focus=patterns
3. **Live-Ball** ‚Üí pick 1-2 from `framework.liveBallFormats` based on remaining minutes
4. **Review + Homework** ‚Üí pick template from `framework.reviewTemplates`, generate one-line homework string
- Minute splits scale with total `durationMin`: movement 12-15%, craft 40-45%, live-ball 35-40%, review 5-10%
- Output validated against `LessonPlan` Zod schema before persist

**Persistence (Vercel KV):**
- Key: `coach-lesson-plan-drafts`
- Value: JSON array of `LessonPlanDraft` (LessonPlan + meta: { id, generatedBy, createdAt, source: 'composer' })
- Read-modify-write on save (acceptable for low concurrency: 3 coaches)
- Cap at 100 drafts (FIFO eviction) to prevent unbounded growth

**Library landing changes:**
- After the 12 curated plans, render a "Generated drafts" section (only if any drafts exist)
- Each draft card has a "Draft" pill in the eyebrow + "Generated by {coach}" attribution

**Detail page handles drafts:**
- Existing `/lesson-plans/[planId]` already does notFound() if not in `plans.json`
- New `/lesson-plans/drafts/[draftId]` reads from KV, falls back to 404
- Both reuse the same `LessonPlanDetailView` component

## Risks & Mitigations

| Risk | Severity | Mitigation |
|---|---|---|
| KV write race condition (two coaches generate simultaneously) | Low | 3 active coaches; collision unlikely. If it ever matters: switch to KV `lpush` (atomic append). |
| KV unbounded growth | Low | FIFO cap at 100 drafts. |
| Composer produces a bad plan (wrong minute splits, missing block) | Medium | Zod schema validation BEFORE persist; bad output ‚Üí 500 (don't store junk). Unit-test the composer with 3-4 sample inputs. |
| Generated plan slug collides with curated `plans.json` ID | Low | Drafts are URL-addressable via `/drafts/[draftId]`, not `/lesson-plans/[planId]`. Different namespace. |
| Form is overwhelming (too many inputs) | Medium | 4 inputs only: Program, Age, Duration, Focus. Theme/notes are inferred. |
| KV unavailable in dev | Low | Same fail-open pattern as `lib/rate-limit.ts` ‚Äî if KV write fails, return helpful error to coach. |
